Запись регистр сведений "Репозиторий объектов интеграции" предназначена для хранения сведений об объекте интеграции (т.н. обработчике объекта).

  • Имя класса объекта - имя класса, по которому обработчик "связывает" сообщение в очереди сообщений с репозиторием  (тип - Строка);
  • Тип интеграции - тип интеграции, в зависимости от которого обработчик либо формирует объект из входящего сообщения, либо формирует исходящее сообщение (тип - перечисление "Типы интеграции": Исходящая, Входящая);
  • Процедура обработки -  процедура обработки объекта на встроенном языке (тип - Строка);
  • Метод хранения - метод, по которому будут храниться данные в сформированном сообщении (тип - перечисления "Метод хранения"):

По ссылке - данные сообщения будут сериализованы в момент отправки сообщения в адаптер ESB;

Сериализация - данные сообщения будут сериализованы сразу же при формировании сообщения и в таком виде попадут в адаптер ESB;

  • Версия - текущая версия записи репозитория (обработчика) (тип - Строка);
  • Идентификатор шаблона - уникальный идентификатор шаблона(тип - Уникальный идентификатор)
  • Наименование - наименование записи репозитория (обработчика) (тип - Строка).
  • Статус - текущий статус обработчика (тип - перечисление "Статусы обработчиков"):

Включен - обработчик активен;

Отключен - обработчик не активен.
