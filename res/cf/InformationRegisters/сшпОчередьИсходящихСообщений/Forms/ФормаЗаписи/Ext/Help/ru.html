Запись регистра сведений "Очередь исходящих сообщений " предназначена для хранения данных исходящего сообщения, сформированного системой, для последующей отправки в ESB.

В форме записи отображается:

  • Идентификатор сообщения - уникальный идентификатор сообщения, сформированного системой (тип - Уникальный идентификатор);
  • Дата регистрации - дата, когда исходящее сообщение было зарегистрировано в очереди (тип - Дата);
  • Объект события - полное имя метаданного, запись которого вызвала формирование исходящего сообщения (тип - Строка)
  • Формат сообщения - формат, в котором записано исходящее сообщение (тип - перечисление "Форматы сообщений": XML, JSON, FastInfoset);
  • Метод хранения - метод, по которому хранятся данные в сформированном сообщении (тип - перечисления "Метод хранения"):

По ссылке - данные сообщения будут сериализованы в момент отправки сообщения в адаптер ESB;

Сериализация - данные сообщения будут сериализованы сразу же при формировании сообщения и в таком виде попадут в адаптер ESB;

  • Статус - текущий статус исходящего сообщения (тип - перечисление "Статусы соощений"). Может принимать значения:

Новое - сообщение только что поступило в систему;

Ожидание обработки - сообщение ожидает обработки;

В обработке - сообщение находится в обработке;

Отсутствует обработчик - для данного класса сообщения не найден обработчик;

Обработано - сообщение обработано;

Ошибка обработки - при выполнении обработчика возникла ошибка;

Обработка отменена - обработка сообщения отменена;

Ожидание отправки - сообщение ожидает отправки (для исходящих сообщений);

Отправлено - сообщение отправлено (для исходящих сообщений);

Отправка отменена - отправка сообщения отменена (для исходящих сообщений).
